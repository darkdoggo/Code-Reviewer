# 国际化（i18n）设计文档

**日期：** 2026-04-22  
**状态：** 已批准  
**目标：** 为 Reviewer Agent 添加中英文切换支持

---

## 概述

实现双语言配置系统，允许用户独立控制：
1. **UI 语言**：界面文本（按钮、标签、菜单等）
2. **LLM 输出语言**：代码审查结果的语言

### 核心需求

- UI 语言和 LLM 输出语言可独立切换
- UI 语言切换器位于顶部导航栏（快速访问）
- LLM 输出语言选择器位于 Settings 页面（高级配置）
- 首次启动时自动检测系统语言
- 使用 react-i18next 实现完整的国际化支持

## 架构设计

### 1. 配置结构

**类型定义更新（`src/shared/types/config.ts`）：**

```typescript
export interface UIConfig {
  theme: 'light' | 'dark' | 'system'
  diffMode: 'unified' | 'split'
  uiLanguage: 'en' | 'zh'  // 新增：UI 语言
}

export interface LLMConfig {
  provider: 'anthropic' | 'openai'
  model: string
  apiKey: string
  baseUrl?: string
  temperature: number
  maxTokens: number
  outputLanguage: 'en' | 'zh'  // 新增：LLM 输出语言
}
```

**默认值：**
- 首次启动时检测系统语言（`navigator.language`）
- 如果系统语言为 `zh-*`，默认 `uiLanguage` 和 `outputLanguage` 都设为 `zh`
- 否则默认为 `en`

### 2. 翻译文件结构

**目录组织：**

```
src/renderer/src/locales/
├── en/
│   ├── common.json      # 通用文本（按钮、标签）

│   ├── settings.json    # Settings 页面
│   ├── review.json      # Review 页面
│   └── errors.json      # 错误消息
└── zh/
```

**命名空间约定：**
- `common:save` → "保存" / "Save"
- `pages:dashboard.title` → "仪表盘" / "Dashboard"
- `settings:llm.apiKey` → "API 密钥" / "API Key"

### 3. i18next 配置

**初始化配置：**

```typescript
// src/renderer/src/i18n.ts
import i18n from 'i18next'
import { initReactI18next } from 'react-i18next'

i18n
  .use(initReactI18next)
  .init({
    resources: {
      en: {
        common: require('./locales/en/common.json'),
        pages: require('./locales/en/pages.json'),
        // ...
      },
      zh: {
        common: require('./locales/zh/common.json'),
        pages: require('./locales/zh/pages.json'),
        // ...
      }
    },
    lng: 'en', // 默认语言，将被配置覆盖
    fallbackLng: 'en',
    ns: ['common', 'pages', 'settings', 'review', 'errors'],
    defaultNS: 'common',
    interpolation: {
      escapeValue: false
    }
  })

export default i18n
```

## UI 组件设计

### 1. 顶部导航栏语言切换器

**位置：** Layout 组件右上角（主题切换按钮旁边）

**视觉设计：**
```
┌─────────────────────────────────────┐
│  Reviewer Agent    🌐 EN  🌙  ⚙️   │
│                     ↓               │
│                  ┌─────────┐        │
│                  │ English │        │
│                  │ 中文    │        │
│                  └─────────┘        │
└─────────────────────────────────────┘
```

**交互行为：**
- 显示地球图标 + 当前语言代码（EN / 中）
- 点击弹出下拉菜单，显示两个选项
- 选择后立即切换 UI 语言，无需刷新页面

**技术实现：**
- 使用 Radix UI 的 `DropdownMenu` 组件
- 图标使用 `lucide-react` 的 `Globe` 或 `Languages`
- 状态通过 Zustand store 管理
- 调用 `i18n.changeLanguage()` 切换语言

### 2. Settings 页面 LLM 输出语言选择器

**位置：** Settings > Model Config 部分（在 Model 选择器下方）

**UI 布局：**
```
Model Config
├── Provider: [Anthropic ▼]
├── Model: [Claude Sonnet 4 ▼]
├── Output Language: [English ▼]  ← 新增
├── API Key: [••••••••••]
└── ...
```

**选项：**
- English - LLM will respond in English
- 中文 - LLM 将使用中文回复

**行为：**
- 修改后需要点击 Save 按钮保存
- 下次审查时生效（不影响已有的审查结果）

**修改 `LLMProvider` 接口：**

```typescript

export interface LLMProvider {

    diff: string, 
    projectContext?: string,
    outputLanguage?: 'en' | 'zh'  // 新增参数
```

```typescript
  diff: string, 
  projectContext?: string, 
  outputLanguage: 'en' | 'zh' = 'en'

  const languageInstruction = outputLanguage === 'zh'

    : ''
  
  const systemPrompt = `You are an expert code reviewer for full-stack applications...${languageInstruction}`
  
  // ... rest of the implementation
}
```

### 4. 输出示例对比

**英文输出：**
```json
{
  "title": "Explicit `any` type detected",
  "description": "Avoid using `any`. Use a specific type or `unknown` instead.",
  "suggestion": "Replace `any` with a specific type"
}
```

**中文输出：**
```json
{
  "title": "检测到显式 `any` 类型",
  "description": "避免使用 `any`。请使用具体类型或 `unknown` 代替。",
  "suggestion": "将 `any` 替换为具体类型"
}
```

**注意：** 静态分析和 Bridge 检查的 issues 保持英文，不受 LLM 输出语言影响。

## 错误处理

### 翻译缺失处理

- i18next 配置 `fallbackLng: 'en'`
- 如果中文翻译缺失，自动回退到英文
- 开发模式下在控制台警告缺失的翻译键

### 语言切换失败

- 如果切换失败，保持当前语言不变
- 显示错误提示（使用当前语言）
- 记录错误日志

### LLM 语言指令失效

- LLM 可能不完全遵守语言指令
- 这是可接受的行为，不影响功能
- 用户可以切换回英文或重新运行审查
